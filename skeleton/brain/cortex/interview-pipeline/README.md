# Recruiting Harness Skill

这是一个招聘流程 Harness。它的核心界面是飞书多维表格：所有候选人信息、流程状态、会议链接、面试稿、录音转写、复盘和测试题进度，都围绕多维表格里的同一条候选人记录管理。

核心目标：用户通过多维表格管理招聘全局；AI 把各类产物回写到表格；候选人只收到必要通知。

## 使用顺序

1. 先跑环境体检：Node、Python、招聘平台 CLI、飞书 CLI、会议平台 CLI、登录态、多维表格字段、会议录制和转写权限。
2. 复制 `docs/recruiting-project-config-template.md`，填写项目配置。
3. 建好飞书多维表格主控台，配置字段和视图，让用户能从表格里查看候选人池、待筛选、待约面、今日面试、面后待复盘、测试题跟进和二面池。
4. 把简历、评分、状态、会议、面试稿、录音转写、复盘都回到同一行候选人记录。
5. 按 `docs/feishu-interview-doc-template.md` 生成每位候选人的飞书面试文档，并把链接回填表格。
6. 配置巡检和手动刷新：多维表格默认每天下午 4 点自动巡检一次；其他时间如需更新，由系统提醒用户手动刷新简历、候选人回复、会议产物或测试题。
7. 每次批量操作后用 `docs/real-flow-audit-checklist.md` 做复核。
8. 候选人沟通只做必要动作：确认是否有空、发送会议链接、必要时发送测试题说明。
9. 面试结束后拉转写、写复盘、更新表格，等用户明确指令再进入下一轮或终态。

## 核心原则

- 飞书多维表格是主控台和单一真相源。
- 飞书面试文档是某一行候选人记录的深度工作页，不发给候选人。
- 任何流程动作完成后都必须回写多维表格；没有回写表格，就视为没完成。
- 真实自动化必须有巡检/刷新游标和日志；默认每天 16:00 巡检多维表格，其他更新按需手动触发。
- 候选人外发消息要克制，不暴露内部轮次、评级、排序、薪资策略和淘汰逻辑。
- AI 负责整理、评分、写稿、复盘和归档；人负责约谁、过不过、是否进入下一轮等关键决策。
- 会议未确认可录制/转写时，不静默约面。
