## 模块简介

招聘面试的固化支线（Harness）。任何面试相关请求都走这条线：

```
简历到达
  → 入表（候选人跟踪表）
  → AI 评估 + 星级
  → 用户挑人 / 约面
  → AI 写面试主持稿
  → 面试后据录音写沟通复盘
  → 用户人工推进状态
```

完整流程见 [`docs/interview-sop.md`](docs/interview-sop.md)（权威源 / 含具体协作人 open_id、表格 ID 等私密配置 —— starter 模板里留占位）。

## 四阶段速记

1. **简历同步**：从来源渠道拉新简历附件 → 下载（docx 转 txt / 无名 PDF 取名） → 写入候选人表 + 上传附件 → AI 写评估 + ⭐星级
2. **面试准备**：每位写**主持稿三段式**（公司介绍开头 + 速览表格 + 追问清单）。追问统一格式「**我从简历看到 X → 所以想问你 Y**」，每个 Y 把"怎么做的 / 你本人具体做了什么 / 结果与贡献"三件事问清（STAR 结构）。约面信息填回表
3. **面试后复盘**：拉对应沟通记录 / 录音 → 写沟通复盘（防附件挂错：复盘要和录音逐字稿匹对再落）
4. **维护**：用户人工推进「面试情况」状态；AI 只读不擅自改判定列

## 主持稿三段式（v0.2.5 提炼）

```
① 公司介绍（开场 1 段，统一文案）
② 候选人速览表格（基本信息 / 经历 / 关键技能 / 风险点 ABCDE 五维度 / 红旗）
③ 追问清单（每条：「我从简历看到 X → 所以想问你 Y」）
```

ABCDE 五维度（可按岗位裁剪）：A 业务理解 / B 技术深度 / C 协作风格 / D 抗压与边界 / E 长期意愿。

## 硬约束

- 只依据简历真实内容评估，**不编造**；异地 / 投错岗 / 薪资倒挂等显式标出
- **不擅自改用户判定的「面试情况」列**（终审在用户）
- 写操作前缀（如 `LARK_CLI_NO_PROXY=1` 或等价）+ @file 路径相对当前目录 —— 具体见 `docs/interview-sop.md`
- 候选人个人信息**不出本支线目录**（不入 gateway / 不入对外可分享 Skill 包）
