脱敏后的招聘 Harness 已命名为 **HireBase Harness**，可直接随 starter 分发。核心定位：**飞书多维表格是招聘主控台和单一真相源**；飞书文档、候选人沟通、会议系统都是多维表格某一行候选人记录的附属对象。

## 首次启用时要填的项目配置

| 配置项 | 说明 |
|---|---|
| 招聘平台 | Boss 直聘 / 邮箱 / 内推 / 其他来源 |
| 飞书多维表格主控台 | 候选人表 base/table 链接和字段 |
| 飞书文档空间 | 面试官文档存放位置 |
| 会议平台 | 腾讯会议 / 飞书会议 / Zoom 等 |
| 巡检配置 | 默认每天 16:00 自动巡检；其他更新手动刷新 |

把这些填进 `docs/recruiting-project-config-template.md` 的项目副本中。不要把 token、cookie、open_id、base_id、候选人姓名、录音、简历放进通用 Skill。

## 运行约束

- 所有流程产物必须回写飞书多维表格同一条候选人记录；没有回写表格，就视为没完成。
- 候选人外发只做必要通知：确认是否有空、发送会议链接、必要测试题说明；不暴露内部轮次、评级、排序、薪资策略、淘汰逻辑。
- 自动巡检默认每天 16:00 一次；简历、候选人回复、会议产物、测试题状态默认由用户手动触发刷新。
- AI 可以建议状态变化，但通过/不通过/进入二面/发 offer 必须等用户明确指令。
