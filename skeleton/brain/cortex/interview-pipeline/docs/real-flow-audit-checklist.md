# 真实流程审计清单

> 这份清单用于在真实招聘项目里跑流程前后做复核。它来自实际招聘流程暴露过的问题：表结构重构、主持稿风格偏差、会议改期不同步、录制/转写缺失、会议链接待补、附件归属错误、候选人外发信息过多。

---

## 1. 启动前

| 检查项 | 必须通过 |
|---|---|
| 招聘平台 CLI | 能读取候选人消息、发送文本、识别候选人回复 |
| 飞书 CLI | 能读写飞书多维表格、创建/读取/更新在线文档、上传附件 |
| 会议平台 CLI | 能创建会议、修改会议时间、开启录制/转写、查询结束会议、拉取转写 |
| 候选人表字段 | 至少具备基础信息、AI 评估、一面、测试题、二面、外联六个分区 |
| 候选人表视图 | 至少具备全部候选人、待筛选、待约面、今日面试、面后待复盘、测试题跟进、二面池、已终结 |
| 在线文档空间 | 有固定的面试稿父节点或文档空间 |
| 标准口径 | 公司介绍、岗位画像、反问预备回答、测试题说明已配置 |
| 样例闭环 | 样例简历能完成解析、入表、评分、生成面试文档、创建测试会议 |

不通过时，不进入真实候选人监听。

---

## 2. 简历入表

核心验收：用户是否能只打开飞书多维表格，就看懂候选人池、优先级、当前状态和下一步动作。如果还需要翻聊天记录或临时文档才能知道下一步，说明流程没有归位。

| 风险 | 审计动作 |
|---|---|
| 重复候选人 | 用姓名、手机号、简历附件名、来源消息时间做去重 |
| 无名简历 | 先读简历内容取真实姓名，原文件名写备注 |
| 批量附件挂错 | 每份附件上传后核对候选人名和记录行 |
| 合并转发未展开 | 单独标记为“待展开”，不能视为已处理 |
| AI 编造信息 | 缺失项写“简历未提及”，不得补全 |
| 候选人表状态误动 | AI 只写评估和建议，不擅自写通过/不通过/进入下一轮 |

---

## 3. 候选人外发

候选人沟通窗口只做必要动作。

允许发送：

```text
你好，请问你 {日期} {时间段} 是否方便做一次线上沟通？预计 {时长} 分钟左右。
如果这个时间不方便，也可以告诉我你方便的时间段。
```

候选人确认后：

```text
好的，那我们约在：
时间：{日期} {开始时间}-{结束时间}
会议：{会议链接}

到时麻烦提前 3 分钟进入会议，保持麦克风和网络可用。谢谢。
```

不要发送：

- 第几轮面试
- 内部评级、优先级、排序
- “你是最匹配/最强候选人”
- 薪资策略和内部预算判断
- 测试题真实考察意图
- 淘汰逻辑或通过暗示
- 面试官内部主持稿链接

---

## 4. 预约会议

| 风险 | 审计动作 |
|---|---|
| 用户只给时间窗口 | 系统按优先级选人，但必须等用户明确“约这批/约 N 位” |
| 候选人未确认 | 状态只能是“待确认”，不能写“已约” |
| 会议时间改了 | 同步改会议平台时间、候选人表时间、候选人外发确认文案 |
| 会议链接待补 | 表格写“待补”，不能假装已完成 |
| 录制/转写未开启 | 不静默约面；必须标红提醒或阻断流程 |
| 会议名称泄露内部信息 | 外发给候选人的文案不带内部轮次、评级、排序 |
| 多场连续会议 | 检查时间槽不重叠，保留缓冲时间 |

推荐内部会议名：

```text
{候选人姓名} - 面试沟通 - {日期时间}
```

外发候选人只看时间和会议链接。

---

## 5. 面试文档

在线面试文档是面试官工作台，不是候选人材料。

创建新文档时：

1. 从候选人表读取简历、评分、风险、会议时间。
2. 按 `feishu-interview-doc-template.md` 生成文档。
3. 写入固定面试稿文档空间。
4. 将文档链接回填候选人表。

修改已有文档时：

1. 先读取在线文档现状。
2. 尊重用户在在线文档里的手改标题、措辞、生僻字和结构。
3. 只做局部替换或追加。
4. 禁止用本地旧稿整篇覆盖在线文档。

风格检查：

| 模块 | 必须有 |
|---|---|
| 文档头部 | 时间、候选人、岗位、内部评级、本轮目标 |
| 开场 | 可照读，2-3 分钟 |
| 候选人速览 | 核心经历、匹配点、风险点、本轮必须验证 |
| 追问 | 每题“我从你简历看到 X，所以想问 Y” |
| 三段式 | 怎么做的 / 你本人具体做了什么 / 结果与贡献 |
| 红旗 | 薪资、地点、到岗、职级降级、测试题未交等前置 |
| 测试题承接 | 作业概览、内部判断、逐条追问 |
| 反问预备回答 | 团队、产品、技术栈、入职节奏 |
| 面试官备忘 | KILL/PASS 信号、本轮决策点、下一步建议 |

---

## 6. 面试进行中

开始前检查：

- 会议链接可打开。
- 候选人已确认参会。
- 录制/云录音已开启。
- 实时转写已开启。
- 面试文档已打开。
- 候选人表该行可定位。

面试中：

- 不共享内部面试文档。
- 红旗问题前置，不拖到最后。
- 测试题说明只说任务和提交方式，不讲内部考察意图。

---

## 7. 面试后

| 风险 | 审计动作 |
|---|---|
| 会议无录制 | 表格标记“无录制/待补”，不能生成伪复盘 |
| 转写拉取失败 | 标记失败原因，安排手动上传或重新拉取 |
| 录音挂错候选人 | 写复盘前先核对候选人姓名、面试时间、说话人 |
| 复盘写错轮次 | 一面复盘写一面字段，二面复盘写二面字段 |
| 测试题未交 | 不进入下一轮，除非用户明确豁免 |
| AI 擅自终态 | AI 只能建议，最终通过/不通过等用户明确指令 |

复盘结构：

1. 总体结论
2. 面试表现
3. 岗位匹配
4. 核心亮点
5. 主要风险
6. 下一步建议

---

## 8. 二面 / 后续轮次

进入下一轮前必须满足：

1. 上一轮会议链接、录音/转写、复盘已回填。
2. 上一轮结果或建议已明确。
3. 测试题如存在，状态已明确。
4. 用户明确说“进入下一轮/约下一次/推进”。

下一轮候选人外发仍只说“下一次线上沟通”，不暴露内部轮次和评价。

---

## 9. 最终交付前复核

每次批量操作后，按候选人逐行检查：

```text
候选人
简历
AI 评估
风险
当前状态
会议时间
会议链接
候选人是否确认
面试文档链接
录制/转写状态
复盘
测试题状态
下一步建议
```

任何一项缺失，都不能宣称该候选人流程已闭环。
